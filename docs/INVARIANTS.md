# STARLINE Ω — Engine Invariants (Frozen)

**Status: POST-MVP FORMALIZATION — do not bypass.**

---

## 1. Frozen core loop

```
Motion → Smoothness → Coherence → Phase → Audio/Visual
```

All future work must **plug into**, **modulate**, or **emerge from** this loop.

**Never bypass it.**

Forbidden parallel progression axes unless reducible to `(v, S, c, φ, R)`:

- redundant “focus”
- “magic energy” / nitro as separate currency
- arbitrary RPG stats

---

## 2. Sacred variables

| symbol | code field | meaning |
|--------|------------|---------|
| `v` | `v` | velocity along arc-length |
| `S` | `smoothness` | differential-geometry quality of control |
| `c` | `coherence` | energetic order / flow state |
| `φ` | `phase` | harmonic phase on the manifold |
| `R` | `sync_R` | multiplayer synchronization order parameter |

Everything else is **derived**.

---

## 3. Dimensional hierarchy (sufficient for years)

| layer | meaning |
|-------|---------|
| Motion | kinematics |
| Smoothness | differential geometry |

Do not add inventory, currencies, or RPG mechanics until expressible **geometrically**.

---

## 4. Audio law

Music is perceptual rendering of state-space:

```
A(t) = f(v, S, c, R, κ)
```

No arbitrary soundtrack switching disconnected from state.

---

## 5. Flow-state thresholds

| state | condition |
|-------|-----------|
| `unstable` | `c ≤ 0.4` |
| `stable` | `c > 0.4` |
| `resonant` | `c > 0.7` |
| `flow` | `c > 0.9` AND `R > 0.8` |

Drives VFX, audio layers, and future progression **without ad-hoc logic**.

---

## 6. Coherence growth (anti-saturation)

```
Δc = α · S · (1 − c)^p · dt  −  β · I · dt
```

with `p > 1` (default `p = 2`). Preserves low-end accessibility and high-end mastery ceiling.

---

## 7. Predictive smoothness

Smoothness uses **anticipation** of upcoming curvature:

```
κ_ahead = κ(s + Δs),   Δs = ds₀ + k_v · v_eff
```

Players prepare corners → rhythm and phrasing emerge.

---

## 8. Tracks are scores

A track is a **musical-geometric composition**, not merely a map.

See `track_score.py` / `TrackScore.lua` for:

- curvature rhythm (percussion)
- straight sustain zones
- torsion / modulation events
- resonance zones (chorus/drop)

---

## 9. Visual style constraints

**Always:** dark ground, sparse geometry, luminous flow, information density > object density.

**Never:** cluttered cities, noisy UI, realistic asphalt sim where geometry becomes unreadable.

---

## 10. Server authority

**Authoritative on server only:**

- `c` (coherence)
- `φ` (phase)
- `R` (synchronization)

Clients may predict for feel; server state wins.

---

## 11. Telemetry mandate

Observe emergent coordination:

- coherence histograms
- corner failure locations (high `I`, low `S`)
- resonance / flow dwell times
- sync clustering
- steering entropy

---

## 12. Noise budget

Every feature adds cognitive / visual / mechanical noise.

Rule: `N_total < N_critical`. If players stop sensing coherence, **remove noise**, not add juice.

---

## 13. Future AI constraint

Any ML / generative / adaptive system optimizes:

```
max(coherence emergence)
```

Not engagement addiction, retention hacks, or monetization friction.

---

## 14. Failure is desynchronization

Failure should:

- sound dissonant
- destabilize geometry subtly
- reduce harmony

Not arcade explosions. Player feels: **“I fell out of sync.”**

---

## 15. Design truth (permanent)

STARLINE Ω is **not fundamentally a racing game**.

It is a **coherent motion system**.

Players are rewarded for smoothness, synchronization, anticipation, and harmonic alignment with curvature.

**Velocity is not granted mechanically. Velocity emerges from coherence.**

**Music is not decoration. Music is the perceptual projection of geometric state.**

The next 80% of quality is **tuning discipline** — instrument-making, not feature sprawl.
